Notebook Journal

List · Home · Form

List page: single column, each topic is a "page entry" with a washi-tape decoration, date, and dashed rose left rule. Generous whitespace.

Homepage: a quiet open-book illustration above a 3-up "recent oppslag" block + a pixel illustration of a leather-bound journal. Pure invitation.

Topic form: notebook-paper background with ruled lines, fields styled as journal blanks, live preview rendered as the next "page entry."

PROS · Cozy, low cognitive load, beautiful idle state. Maps well to a "gradually growing knowledge collection."

CONS · Slow to scan when you have 100+ topics. No category structure. Feels personal — maybe too personal for a shared org tool.

Field Guide

List · Home · Form

List page: sidebar of categories (KAPITLER) on the left, grid of topic cards on the right. Each card: pixel sprite + title + version tag. Featured cards get rose border.

Homepage: hero with table-of-contents tiles (Hjerte/Lunge/Trauma/...), each with a representative pixel illustration. Below: "siste oppdateringer" feed.

Topic form: two-pane: chapter selector + form fields on left, live "Field Guide entry" preview card on right with the sprite the editor picked.

PROS · Scales to 100+ topics. Clear domain structure (matches how paramedics learn — by system: hjerte, lunge, trauma). Closest in DNA to current TopicForm split layout.

CONS · Sidebar adds chrome. More to build (chapter taxonomy, sprite library per topic).

Botanical Index

List · Home · Form

List page: alphabet-tabbed dividers on top, list rows with tiny pixel icon + title + summary + meta. Dotted leader lines between rows. Reference-book density.

Homepage: single hero ("Index of paramedic procedures") + recent updates as inline rows + a small pixel illustration of a botanical-style title plate.

Topic form: compact left form, right preview rendered as two adjacent index rows showing the new topic in alphabetical context (with neighbors).

PROS · Highest information density. Rewards experts. The index/leader-lines aesthetic matches "reference book" intuition perfectly.

CONS · Less inviting at small N (looks empty when you only have 5 topics). Pixel sprites at 12×12 do less visual work.
